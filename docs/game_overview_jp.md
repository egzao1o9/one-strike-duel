# One Strike Duel 現状まとめ

## 概要

`One Strike Duel` は、少ない手数で相手の読みを外し、一撃を通すことを目指す対戦ゲームです。  
カードゲームでありながら、感覚としては「読み合い重視の間合い勝負」に近く、単純な打点の押し付けだけでは勝ちきれない構造を目指しています。

現状は、以下の要素でゲームが構成されています。

- ドラフトで12枚デッキを作る
- 対戦中は手札・公開情報・捨て札を見ながら判断する
- `battle / control / blessing` の3種のカードを使い分ける
- バトルでは、何枚伏せるか、どこで止まるか、どの順番で出すかが重要になる

## 現在の基本ルール

### デッキ

現状の対戦用デッキは12枚です。

- `Common 5`
- `Uncommon 5`
- `Rare 2`

加えて、全プレイヤーは初期デッキとして以下の3枚を持ちます。

- `攻撃`
- `防御`
- `ステップ`

この初期デッキをベースに、ドラフトで残りのカードを加えて最終デッキを作ります。

### ドラフト

現状のWeb版では、マーケット式のドラフトを採用しています。

- 場には `Rare / Uncommon / Common` がそれぞれ3枚ずつ公開される
- 各手番で
  - 場から1枚取る
  - 同レアリティ山札のトップを取る
  のどちらかを選ぶ
- 公開カードを取った場合は、同レアリティのカードが補充される
- トップ取りは取得するまで相手に見えず、取得後は自分だけが把握する

このドラフトにより、

- 安定して強い公開札を取る
- あえて非公開で拾って相手に情報を渡さない
- 相手に見えている公開情報から、相手デッキを読む

という駆け引きが生まれます。

### 手札

- 初期手札は4枚
- 1ターン目は通常ドローなし
- 2ターン目以降は、ターン開始時に最大4枚まで引く
- 手札上限は6枚
- 使わなかった手札は持ち越す

このため、1ターン目は「限られた4枚でどう動くか」が重く、2ターン目以降は捨て札・山札再利用・残り札の推測が効いてきます。

### フェーズ

現状の対戦は概ね以下の流れです。

1. `Mulligan`
2. `Control`
3. `Battle`
4. `Reveal`
5. `Blessing`
6. `Result / Next Turn`

#### Mulligan

- 不要な手札を選んで引き直す
- 初手の質を整える

#### Control

- `control` カードや `blessing` カードを使う
- control はそのターンの補助
- blessing は場に残り、特定タイミングで使える常備札

#### Battle

- 先手から交互に `set / set_pass / pass` を行う
- `battle` カードだけでなく、`control` カードをバトルに伏せることも可能
- 自分の伏せ枚数は常に「相手の伏せ枚数 + 1枚」まで

#### Reveal

- 両者が止まったらカードを1枚ずつ公開
- 公開順に効果を処理
- 途中で `on_reveal` 系や割り込み確認が入る

#### Blessing

- 任意使用型の blessing があれば、最終計算前に使うか確認する

#### Result

- 最終 `A / B / S` を比べて勝敗判定
- Draw や決着なしなら次ターンへ

## カードの役割

### Battle

直接 `Attack / Block / Speed` を持つカードです。  
最終的なバトルの土台になります。

役割の例:

- 高打点で押し切る
- 速さで先制する
- 防御を厚くして受ける
- 受けながら少しだけ攻める

### Control

そのターンの補助や妨害を担います。  
単純バフ・デバフだけでなく、

- 公開情報の獲得
- 1枚目公開セットの強制
- blessing 干渉
- 山札操作
- 捨て札操作
- 特定カード選択

のような、情報戦と盤面制御を担当します。

### Blessing

場に残る特殊カードです。  
1試合中に1枚だけ置ける想定で、使い切り型や常在型があります。

役割の例:

- 必要なタイミングで攻撃/防御/速度を補う
- 相手の強い攻めを止める
- 公開時に相手の伏せを暴く
- 相手の blessing を封じる/裏返す

## このゲームの駆け引き

### 1. ドラフト時点で読み合いが始まっている

このゲームは、対戦開始前から読み合いが始まっています。

- 公開で取ったカードは相手に見える
- 非公開で取ったカードは相手に見えない
- ただし公開カードの傾向から、非公開側の色もある程度読める

つまり、

- 何を取ったか
- 何を見せたか
- 何を隠したか

そのすべてが情報になります。

### 2. 何枚伏せるかが重要

このゲームでは、強いカードを持っているだけでは足りません。  
「どこまでカードを出すか」が非常に重要です。

典型的な判断はこうです。

- 1枚で様子を見る
- 2枚で押しにいく
- 3枚目を相手に使わせる
- あえて止まって次ターン勝負にする

多く出せば数値は伸びますが、その分リソースを消耗します。  
少ない枚数で受ける、少ない枚数で勝つ、相手より1枚多く出す、という判断自体が勝負になります。

### 3. 公開順にも意味がある

カードはまとめて処理されるのではなく、1枚ずつ公開して処理されます。  
そのため、

- どのカードを先に置いたか
- どのカードを後ろに置いたか
- 相手の何枚目に何を合わせるか

が意味を持ちます。

これは今後さらに重要になる部分で、

- 「先に公開されたカードだけに効く」
- 「公開済みの自分のカードを対象にする」
- 「公開済みの相手カードを見てから使う」

ような効果が増えるほど、順番の重みが増します。

### 4. 情報の取り方が勝率を左右する

`探り` のような効果や、公開ピック、捨て札、場のカードから相手の構成を読むことができます。

例えば、

- 相手が攻撃寄りか
- 防御寄りか
- 速度寄りか
- blessing 軸か
- control 多めか

を推測できれば、手札の切り方が変わります。

このゲームでは「完全に読む」のではなく、

- だいたい合っている
- でも外れることもある

くらいの不確実さが面白さになります。

### 5. Draw や決着なしも戦略の一部

勝ちにいくことだけが目的ではありません。  
守る側は、そのターンをしのいで次ターンに有利を作ることができます。

そのため、

- このターンは止めるだけでよい
- 相手の強札を吐かせれば十分
- 自分の blessing を温存して次ターン勝つ

という発想が成立します。

## 面白いポイント

### 一撃必殺感と、細かい読み合いが両立している

最終的には一撃を通すゲームですが、そこに至るまでの判断はかなり細かいです。

- 何を取るか
- 何を見せるか
- 何を隠すか
- 何枚伏せるか
- どこで止まるか
- どの順に置くか
- blessing をいつ切るか

この積み重ねが、最後の一撃の質を決めます。

### 数字の押し付けだけで終わらない

単純に高い `A / B / S` を積めば勝てるわけではありません。

- 速いけど打点が足りない
- 守れるけど決めきれない
- 1枚強いけど相手に枚数差を付けられる
- blessing を読まれる

というズレが起きるので、常に「今この局面に必要な数値」を考える必要があります。

### 公開情報が多いのに、完全情報ではない

相手の公開札、捨て札、使用済み札から多くを読めます。  
それでも非公開札は残るため、完全な計算ゲームにはなりません。

この

- 読める
- でも読み切れない

感覚が、このゲームの大きな面白さです。

### アナログ化やオンライン化にも向く構造

ルールの本質は

- 公開情報
- 非公開情報
- 伏せ
- 順番
- タイミング確認

にあります。

そのため、将来的に

- ボードゲーム化
- オンライン対戦化
- スマホアプリ化

を考えるうえでも芯がぶれにくい構造になっています。

## 現状の課題

### 認知負荷が高い

このゲームは情報量が多く、初見ではかなり重く感じやすいです。

特に負荷が高いのは:

- ドラフト中に相手の公開情報も見る必要がある
- 伏せ枚数と順番も考える必要がある
- blessing や control の割り込みがある

そのため、UI での補助やルールの整理は今後も重要です。

### 効果の実装と整理はまだ発展途上

現状はWeb版でかなり多くの処理が入ってきていますが、今後も

- 対象選択
- 山札操作
- 捨て札操作
- 公開順依存
- 任意使用タイミング

の実装を丁寧に積み上げる必要があります。

## 今後の展望

短期的には、

- CPU戦を安定させる
- 効果実装を広げる
- UI の認知負荷を下げる

が優先です。

中期的には、

- 人間同士で遊びやすい体験を整える
- リプレイやログ再生を整備する
- オンライン対戦へ繋がる構造を固める

ことが重要になります。

長期的には、

- itch.io のプロトタイプ公開
- 面白さ検証
- ボードゲーム化 / アプリ化

へ繋がる土台にしていく想定です。

## ひとことで言うと

`One Strike Duel` は、

**「情報を集め、相手を読み、限られた手数の中で一撃を通すゲーム」**

です。  
派手な連鎖や大量展開よりも、1枚の質、1枚の順番、1ターンの判断が重いゲームです。
